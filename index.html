HumanizerLearn more...

This is a crowdfunding application. The request was that the user can see all events or make a donation without having an account (anonymously). If someone wants to create an event or have his own event or donation statistic, he/she must be logged in. Logged in user can change the password, or email, or any other part of an account.

Humanizer was a four-week internship project in Consulteer's office in Zrenjanin, Serbia. I worked with other three colleagues, also interns in frontend development. My responsibility was to deal with the sign-up, log-in, log-out user, and adding functionality to a user account (update account, user name, date of birth, picture...). I also managed multi-language with ngx- translate. The app is made in Angular 6, with Bootstrap4 for style, for backend we used Firebase real-time database. For team collaboration, we used Bitbucket, Youtrack, and Slack.

SnippetsAppLearn more...

This was a four-weeks internship project in Levi9 office in Zrenjanin, Serbia. The four of us, interns in frontend development, we're working on an application that helps developers to create, save, update their code snippets, and see the snippets of others if they are public. Snippets can also be private and only author can see those. The user must sign-up and create an account to use the app. Admin can block any user if necessary, erase snippet... The app don't have to be responsive for small devices (like phone), just for tablets.

I was working in every part of the application. I deepened my knowledge and polished my skills in Angular6, Bootstrap4, Firebase, Git, and Trello.

DoctorsOfficeAppLearn more...

This application is build as study project. Main goal was to learn how MVC and Entity works, and resolve all kind of problems that comes along... In this project I learned everything I currently know about ASP.NET MVC 5 and Entity Framework 6

This application is for doctors office records. Someone must be registrated and loggedin user to see records about doctors, patients or examinations records. User must be email confermed. Also forgot password functionality is enabled. All entities (doctor, patient and Examination) can be created, edited or deleted. There is possibility of image upload for doctor, or multiple files upload for examination. The app is hosted on Azure and uses its sql database.
